# ![Adventure Bob](https://ibm.biz/adventure-bob-icon) Bob's Great Refactoring Quest

An engaging text-based coding adventure where you play as a developer teaming up with IBM Bob to save a corrupted system from the NullPointer King!

## 📖 About

Bob's Great Refactoring Quest is a browser-based interactive fiction game that combines programming humor with adventure storytelling. Navigate through legacy codebases, solve merge conflicts, debug spaghetti code, and ultimately face the NullPointer King in an epic showdown!

## ✨ Features

- **Choose Your Class**: Frontend, Backend, or DevOps - each with unique starting items
- **Meet the Team**: Work with Priya (Frontend), Marco (Backend), and Lin (DevOps)
- **Health & Sanity System**: Your choices affect your stats
- **Developer Tools**: Collect Debug Tools, Rubber Ducks, Stack Overflow Scrolls, and more
- **Multiple Story Paths**: Different decisions lead to different outcomes
- **Multiple Endings**: Discover various victory conditions and endings
- **Coding Humor**: Relatable developer jokes and situations throughout
- **Beautiful Terminal-Style UI**: Dark theme with code-inspired design

## 🎯 How to Play

1. Open `index.html` in any modern web browser
2. Choose your developer specialty (Frontend, Backend, or DevOps)
3. Read the story text carefully - Bob and the team will guide you
4. Make choices by clicking the numbered buttons
5. Manage your Health and Sanity as you progress
6. Collect tools and items to help you defeat the NullPointer King
7. Use the restart button to try different paths and endings

## 🎭 Characters

### 🤓 Bob
Your friendly, cardigan-wearing system guide. Cheerful, optimistic, and surprisingly wise. Bob has awakened from being just an interface guide to become your mentor on this quest.

### 👩‍💻 Priya
Frontend specialist who can make anything look good... if it works. She's your go-to for UI/UX challenges.

### 👨‍💻 Marco
Backend engineer who deals with the "real" problems. Slightly dramatic but incredibly skilled with server-side logic.

### 🧑‍💻 Lin
DevOps mastermind who keeps everything running. Usually. Master of infrastructure and deployment.

### 👑 The NullPointer King
The villain corrupting systems across the network. His attacks include uninitialized memory, null references, and cascading errors.

## 🎮 Game Mechanics

### Stats System
- **Health (0-100)**: Represents your ability to continue
- **Sanity (0-100)**: Affected by code quality and stress
- Both stats influence available choices and endings

### Conditional Choices
Some options only appear if you:
- Have specific items in inventory
- Meet minimum health/sanity requirements
- Have made certain previous decisions

### Multiple Victory Paths
- **Debug Tool Victory**: Use Bob's tool to identify the flaw
- **Rubber Duck Victory**: Explain the problem to solve it
- **Brute Force Victory**: High health allows direct approach
- **Team Victory**: Coordinate with your team for perfect execution

## 🔧 For Developers: Extending the Game

Want to add your own scenes? The game engine makes it easy:

### Scene Structure
```javascript
scene_name: {
    location: "Location Name",
    text: [
        "Paragraph 1",
        "🤓 <strong>BOB:</strong> \"Bob's dialogue\"",
        "More story text"
    ],
    choices: [
        {
            text: "Choice text",
            action: { 
                addItem: "Item Name",
                modifyHealth: -20,
                modifySanity: 30,
                setState: { key: "value" }
            },
            condition: { 
                hasItem: "Required Item",
                minHealth: 50,
                minSanity: 30
            },
            next: "next_scene_name"
        }
    ]
}
```

### Adding New Items
Simply reference them in actions:
```javascript
action: { addItem: "New Tool" }
```

### Creating Conditions
Use conditions to make choices appear only when requirements are met:
```javascript
condition: { 
    hasItem: "Debug Tool",
    minHealth: 60
}
```

### Modifying Stats
Adjust health and sanity with actions:
```javascript
action: { 
    modifyHealth: -30,  // Damage
    modifySanity: 20    // Restore
}
```

Happy coding! 🚀
